# Seguridad

No publiques exploits, tokens, datos personales ni detalles que faciliten abuso en un issue.

Usa **Security > Report a vulnerability** en GitHub para enviar un reporte privado. Incluye versión,
impacto, superficie, pasos mínimos de reproducción y mitigación conocida. El mantenedor confirmará
recepción cuando tenga capacidad; este proyecto comunitario no promete un SLA.

Se soporta la release menor más reciente. Una versión npm publicada no se reescribe: una corrección usa
un patch nuevo, y la versión vulnerable puede marcarse deprecated. Credenciales expuestas se revocan; no
se consideran reparadas solo por borrarlas del historial visible.
